# 주도주 스크리너 서비스 워크플로우

## 서비스 운영 요건

### 환경 조건
| 항목 | 요건 | 현재 상태 |
|------|------|----------|
| PC 가동 | 장중(08:50~15:40) 필수 | ✅ scheduler.py 데몬으로 운영 중 |
| Flask 서버 | 5001 포트 상시 가동 | ✅ 가동 중 |
| ngrok 터널 | Cloudflare → Flask 연결 | ✅ 고정 도메인 |
| KIS API 키 | 모의/실전 구분 | ✅ 모의투자 키 확보 |
| 인터넷 | 안정적 연결 필수 | ✅ |

| 항목 | 모의투자 | 실전투자 |
|------|---------|---------|
| Rate Limit | 초당 5건 | 초당 20건 |
| 토큰 유효기간 | 24시간 | 24시간 |
| WebSocket 구독 | 41종목 | 41종목 |
| 장중 데이터 | 실시간 | 실시간 |

## 서비스 워크플로우

### 일일 타임라인

```
08:50  [자동] 토큰 사전 발급
         └─ scheduler.py에서 KIS 토큰 갱신
         └─ 메모리에 캐시 (24시간 유효)
         └─ 실패 시 3회 재시도 → 텔레그램 알림

09:00  [자동] 스크리너 폴링 시작
         ├─ 장 시작 감지 (KRX 휴장일 체크)
         ├─ 3초 간격 폴링 루프 시작
         └─ 첫 결과 ~0.3초 (병렬) ~ 2초 (투자자 포함)

09:00  [사용자] 앱에서 /dashboard/kr/screener 접속
~        ├─ useAutoRefresh(3000) 자동 갱신
15:30    ├─ 등급별 필터 (S/A/B)
         └─ 종목 클릭 → 차트/상세 분석

09:00  [자동] S등급 발생 시 알림
~        ├─ 앱 내 벨 알림 (즉시)
15:30    ├─ 텔레그램 알림 (S등급만)
         └─ 중복 알림 방지 (동일 종목 5분 쿨다운)

15:30  [자동] 폴링 중단
         ├─ 최종 결과 JSON 저장
         └─ data/screener_leading_YYYYMMDD.json

15:30  [자동] 일별 리포트 생성
         ├─ S/A등급 종목 마감가 대비 수익률
         └─ 히스토리 축적 (성과 추적용)
```

### 에러 대응 플로우

```
[KIS API 오류]
  ├─ 401 Unauthorized → 토큰 자동 재발급 (1회)
  ├─ 429 Rate Limit   → 1초 대기 후 재시도 (최대 3회)
  ├─ 500 Server Error → 5초 대기 후 재시도
  ├─ Timeout (10초)    → 다음 사이클에서 재시도
  └─ 3회 연속 실패     → 텔레그램 알림 + 폴링 중단

[네트워크 오류]
  ├─ ngrok 끊김       → 앱은 static fallback 사용
  ├─ Flask 다운       → watchdog이 자동 재시작
  └─ PC 꺼짐         → 서비스 불가 (클라우드 이전 필요)

[장외 시간]
  ├─ 폴링 자동 중단
  ├─ 마지막 결과 캐시 유지
  └─ 앱에서 "장 마감" 표시
```

### 데이터 저장 구조

```
data/
  ├─ screener_leading_latest.json     # 최신 결과 (3초마다 갱신)
  ├─ screener_leading_20260324.json   # 일별 아카이브
  └─ kis_token_cache.json             # 토큰 캐시 (24시간)
```

## 기존 시스템과의 통합

### scheduler.py 연동
```python
# 장중 스크리너 스케줄
schedule.every(3).seconds.do(run_screener_cycle).tag('screener')

# 조건: 장중만 실행
def run_screener_cycle():
    if not is_market_open():
        return
    results = screener.run()
    save_results(results)
    check_alerts(results)  # S등급 알림
```

### Flask 엔드포인트 연동
```
기존:
  /api/kr/jongga-v2/latest    → 종가베팅 (장 마감 후)
  /api/kr/vcp-enhanced        → VCP 시그널 (장 마감 후)
  /api/kr/market-gate         → 시장 게이트 (실시간)

신규:
  /api/kr/screener/leading    → 주도주 스크리너 (장중 실시간)
```

### 프론트엔드 연동
```
사이드바:
  KR Market

    ├─ VCP Signals
    ├─ 종가베팅
```

## 리스크 및 고려사항

### 모의투자 vs 실전투자
| 항목 | 모의투자 | 실전투자 |
|------|---------|---------|
| Rate Limit | 초당 5건 (병목) | 초당 20건 (여유) |
| 데이터 정확도 | 동일 | 동일 |
| 투자자 데이터 | 장 종료 후 제공 | 장 종료 후 제공 |
| 계좌 필요 | 모의계좌 | 실전계좌 |

**주의**: 투자자 매매(외인/기관) 데이터는 장중에는 지연 제공.
장중 실시간 수급은 프로그램매매 데이터로 대체 가능.

### API 호출 최적화 (모의투자 초당 5건 한도)
```
현재: 순차 18건 → 6.8초 (초당 ~2.6건)  ← 모의투자 한도 내
병렬: 최대 5건/초 → 투자자 15건 = 3초
전체 예상: ~3.5초/사이클 (모의투자)

실전투자 전환 시:
병렬: 최대 20건/초 → 투자자 15건 = 0.75초
전체 예상: ~1.5초/사이클
```

### KRX 휴장일 처리
- pykrx `get_nearest_business_day_in_a_week()` 활용
- 또는 KIS API `chk_holiday` 엔드포인트 사용
- 토/일/공휴일 자동 스킵

### 토큰 갱신 전략
```
1. 서버 시작 시 발급 (또는 캐시 로드)
2. 23시간 후 자동 갱신 (만료 1시간 전)
3. 401 응답 시 즉시 재발급
4. 재발급 실패 시 5분 후 재시도 (최대 3회)
```
